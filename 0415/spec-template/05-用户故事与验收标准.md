## 2. 详细用户故事与验收标准

### US-001 上传与解析研报

**故事描述**：作为分析师，我想要上传PDF研报，以便于系统帮我自动解析关键信息。

**验收标准**：

| 验收项 | 标准 |
|--------|------|
| AC-001-1 | 支持拖拽上传和点击选择两种方式 |
| AC-001-2 | 支持PDF和HTML格式，单文件≤50MB |
| AC-001-3 | 支持批量上传，一次最多10个文件 |
| AC-001-4 | 上传后自动触发解析，显示解析进度 |
| AC-001-5 | 解析完成后展示提取的关键信息卡片 |
| AC-001-6 | 解析失败时显示错误原因，支持重试 |

**业务流程**：
```
用户拖拽PDF到上传区 → 系统校验格式大小 → 开始上传 → 
上传完成自动解析 → 显示解析进度 → 完成展示结果
```

## 3. 用户旅程地图

### 核心场景：快速分析研报

```
【发现】分析师收到新研报邮件
   ↓
【上传】打开投研助手，拖拽PDF上传（1分钟）
   ↓
【解析】系统自动解析，查看提取的关键信息（30秒）
   ↓
【阅读】快速浏览原文，右侧查看股票行情
   ↓
【提问】针对疑问点提问，获取精准答案（2分钟）
   ↓
【对比】查找历史研报，生成对比表（1分钟）
   ↓
【输出】收藏研报，导出对比结果
```

**总耗时：约5分钟**（传统方式需2-3小时）
