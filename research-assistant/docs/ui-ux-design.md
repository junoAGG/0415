## 一、需求分析

### 1. 目标用户画像

#### 用户 A：投研分析师
- 高频使用对比分析能力，关注多份研报在评级、财务预测、核心观点上的差异
- 需要快速抽取"共识 / 分歧 / 投资建议"
- 偏好结构化、可导出、可复盘的结果呈现

#### 用户 B：基金经理 / 研究员
- 更关注问答效率，希望通过自然语言快速定位观点和依据
- 需要清晰知道当前问答基于哪些研报样本
- 希望保留多轮会话，并可按主题组织历史记录

#### 用户 C：投研助理 / 初级研究员
- 需要明确引导，不希望面对空白页或纯文本结果
- 对错误状态、加载状态、输入提示更敏感
- 更依赖系统提供默认维度、推荐问题、空状态说明

---

### 2. 核心使用场景

1. **多篇研报快速横向对比**
   - 选 2~5 份已完成研报
   - 选择"公司 / 行业 / 自定义"对比模式
   - 输出结构化分析结论并导出

2. **基于样本集合发起 AI 问答**
   - 基于全部研报或已选研报发起提问
   - 流式接收回答
   - 查看引用来源，继续追问

3. **管理多个研究主题会话**
   - 按股票 / 行业 /策略主题切换会话
   - 搜索历史会话
   - 对会话打标签，提升复用效率

4. **输出分析结果用于汇报或归档**
   - 导出为 Markdown / PDF / CSV / TXT
   - 将对比结论、维度明细、引用来源沉淀为可传播内容

---

### 3. 关键用户痛点

1. **对比结果呈现层级弱**
   - 当前 `compareResult` 仅以文本列表展示
   - 用户难以快速扫描"关键差异、共识、建议、维度深挖"

2. **流式问答反馈不完整**
   - 已有 chunk 逐步追加，但缺少更明确的"正在生成中"心智反馈
   - 无停止生成 / 重试 / 失败恢复
   - 无统一错误区和恢复动作

3. **会话管理能力不足**
   - 仅支持新建、切换、删除
   - 无搜索、标签、分组、筛选、分页
   - 会话规模上来后可管理性差

4. **状态设计不统一**
   - 空状态、加载状态、错误状态样式分散
   - 成功反馈只依赖 message，缺少页面内反馈

5. **页面职责耦合**
   - `Analysis.tsx` 604 行，配置、结果、问答、会话都在一个文件
   - 后续维护与扩展成本高

---

## 二、现状分析

## 1. 当前页面结构

当前 `Analysis.tsx` 采用三栏布局：

- 左栏：分析配置
- 中栏：对比结果 / AI问答主区
- 右栏：分析历史 / 会话管理

当前网格定义来自 `App.css`：

```css
.analysis-layout {
  display: grid;
  grid-template-columns: 320px minmax(720px, 1fr) 280px;
  gap: 12px;
}
```

### 现状优点
- 已有稳定的三栏工作台范式，和全局工作台一致
- 主题变量完整，适合继续扩展
- AI问答已完成 SSE 流式基础能力
- 已有 `typing-cursor`/`blink` 动画基础

### 现状问题
- 左栏信息密度过高，研报选择和模式配置耦合
- 中栏对比结果缺少信息分层与可视化
- AI 问答侧边栏缺少检索与组织能力
- 结果区无工具栏，导出、重试、筛选都缺失
- 空 / 加载 / 错误交互没有统一组件化方案

---

## 三、设计目标

1. **提升分析效率**：让用户 5 秒内看懂对比结果重点
2. **强化流式问答体验**：让"生成中、停止、失败、重试"形成完整闭环
3. **增强历史沉淀能力**：让会话和分析结果具备管理、筛选、导出能力
4. **保证工程可实施性**：基于现有 Ant Design + Tailwind + CSS 架构逐步落地
5. **保持品牌一致性**：沿用金融科技浅蓝金色设计语言

---

## 四、设计方案

## 方案 A：专业工作台增强版（推荐）

### 设计理念与风格
以"专业投研工作台"为核心，延续当前三栏结构，但强化信息层次与结果可视化。
关键词：**稳重、专业、结构化、高密度可扫描**。

### 布局结构描述

#### 对比分析模式
- **左栏 320px：配置中心**
  - 模式切换
  - 对比类型选择
  - 分析维度选择
  - 已选研报概览
  - 研报选择列表（支持搜索 / 快捷筛选）

- **中栏 自适应：结果主工作区**
  - 顶部 Hero + 操作栏
  - 结果摘要 KPI 区
  - 共识 / 分歧双列卡片
  - 投资建议卡片
  - 维度明细 Accordion
  - 来源研报卡片区

- **右栏 280px：历史与导出辅助区**
  - 最近分析记录
  - 导出入口
  - 快捷操作（复制摘要 / 新建问答）

#### AI问答模式
- **左栏 280~320px：会话侧边栏**
  - 搜索框
  - 标签过滤条
  - 会话分组（今天 / 最近7天 / 更早）
  - 会话列表
  - 新建会话按钮

- **中栏：对话主区域**
  - 顶部上下文范围条
  - 对话流
  - 引用来源展开区
  - 输入框 + 发送 / 停止 / 重试

- **右栏 280px：上下文与快捷操作**
  - 当前引用研报范围
  - 推荐问题
  - 最近导出记录 / 当前会话元信息

### 配色方案
沿用 `fintech-theme.ts`：

- 主背景：`#f3f8ff`
- 面板背景：`rgba(255, 255, 255, 0.92)`
- 主文本：`#12233f`
- 次文本：`#57708f`
- 弱文本：`#7d92ad`
- 主要蓝色：`#2b6cb0`
- 深蓝：`#183b70`
- 金色强调：`#f4cf73`
- 成功：`#18a16d`
- 错误：`#cf4b5a`
- 警告：`#da8b2f`

#### 语义色建议
- 共识卡片：蓝色系 `#dff1ff` / `#2b6cb0`
- 差异卡片：金色系 `#fff3cb` / `#d3a63d`
- 建议卡片：深蓝 + 金色描边
- 错误反馈：浅红背景 `#ffe8eb` + `#cf4b5a`

### 字体与排版规范
基于主题：
- 标题：24px / 700
- 分区标题：15px / 700
- 卡片标题：14px / 600
- 正文：13px / 400
- 辅助说明：11~12px / 400
- 数值重点：18px / 700

### 关键交互说明
- 对比维度采用"可勾选卡片"而不是仅 checkbox
- 对比结果区支持"摘要 / 维度 / 来源"页内锚点切换
- AI 生成中显示打字机文本、闪烁光标、停止按钮
- 失败后在消息尾部显示"重试本次回答"
- 会话项 hover 出现更多操作：重命名 / 加标签 / 删除

## 五、推荐方案

### 推荐：方案 A「专业工作台增强版」

### 推荐理由
1. 与当前三栏工作台结构一致，迁移成本最低
2. 最能解决当前用户的核心痛点：结果弱结构、会话管理弱、状态设计不统一
3. 可在不大改数据结构的前提下先完成大部分体验升级
4. 对开发团队最友好，适合分阶段落地

## 六、布局架构优化

## 1. 对比分析面板重构

### 当前问题
左栏同时承载模式、维度、研报列表，用户需要不断滚动。
中栏又没有明确结果层级。

### 优化建议

#### 左栏信息分组
将左栏拆成 4 个明确区块：

1. **模式与类型区**
   - 模式切换：对比分析 / AI问答
   - 对比类型：公司 / 行业 / 自定义

2. **维度选择区**
   - 默认勾选推荐维度
   - 增加"全选 / 清空 / 恢复推荐"快捷操作

3. **已选样本概览区**
   - 展示已选数量
   - 显示已选公司 chip
   - 显示是否满足"至少 2 份"条件

4. **研报选择区**
   - 顶部搜索框
   - 次级筛选：券商 / 评级 / 公司
   - 列表支持批量选择

#### 中栏结果空间分配
建议结构：

- 顶部结果头部：96~120px
- 摘要区：120px
- 共识 / 分歧区：160px
- 维度明细区：自适应填充
- 来源区：底部折叠展示

#### 推荐比例
桌面端：
- 左栏 320px
- 中栏 `1fr`
- 右栏 280px

## 七、交互设计优化

## 1. 对比结果卡片化展示方案

### 结果层级
1. **顶层摘要**
   - 标题：`A / B / C 对比分析`
   - 元信息：对比类型、样本数、时间
   - 操作：导出、复制摘要、重新分析

2. **摘要 KPI 卡片**
   - 维度数
   - 共识数
   - 核心差异数
   - 建议条数

3. **共识 / 差异双列卡片**
   - 左列：共同点
   - 右列：差异点
   - 每项支持复制

4. **投资建议卡片**
   - 单独高亮
   - 每条建议带优先级标签

5. **维度分析 Accordion**
   - 每个维度一个折叠面板
   - 标题显示维度名 + 摘要 + 条目数

---

## 2. 维度分析结果的折叠/展开交互

### 建议交互
- 默认展开前 2 个最关键维度
- 其余折叠
- 支持"全部展开 / 全部收起"

### 展开后的内容结构
- 顶部摘要
- 详细要点列表
- 快捷操作：复制本维度、导出本维度、基于此维度继续提问

---

## 3. 流式问答的打字机效果

### 打字反馈分层
1. **请求发起后 0~300ms**：立即插入"AI 正在分析问题..."占位消息
2. **流式返回中**：chunk 增量渲染 + 末尾闪烁光标 + "生成中"状态条
3. **完成后**：光标消失、显示来源数、提供"重试 / 复制 / 导出回答"

---

## 4. 流式问答中断按钮与重试按钮设计

### 停止生成按钮
- 位置：输入框右侧发送按钮位置在生成中切换为"停止生成"
- 视觉：危险弱化态（背景 `#fff1f2`，字色 `#cf4b5a`）
- 行为：调用 abort，保留已生成部分，显示"已停止生成"

### 重试按钮
- 出现时机：网络错误、AI服务报错、用户停止后、空回答
- 位置：错误消息卡片底部或最后一条AI消息操作区
- 行为：复用上一条问题，保留会话上下文

---

## 5. 数据导出按钮位置与格式选择设计

### 推荐位置
- 对比分析：中栏结果头部右上角
- AI问答：单条回答操作区 + 会话级导出

### 下拉菜单格式
- Markdown（推荐）
- PDF
- TXT
- CSV（对比结果适用）
- 复制到剪贴板

## 八、状态设计

## 1. 空状态设计

| 场景 | 标题 | 说明 | 操作 |
|------|------|------|------|
| 无研报可对比 | 暂无可对比研报 | 请先完成至少2份研报解析 | 去研报管理 / 刷新 |
| 已选不足2份 | 等待对比样本 | 至少选择2份已完成研报 | 全选已完成研报 |
| 无会话 | 开启第一轮研究问答 | 发送问题后将自动创建会话 | 推荐问题chip |
| 无对话历史 | 欢迎卡片 | 当前知识范围与推荐提问模板 | 推荐问题 |

## 2. 加载状态设计

- **对比分析**：骨架屏（标题 + 3个摘要卡 + 双列卡 + 维度折叠项）
- **会话列表**：6条列表skeleton
- **问答流**：动态点阵占位，超3秒显示"模型响应较慢"

## 3. 错误状态统一设计

统一错误卡结构：图标 + 标题 + 描述 + 错误码(可折叠) + 操作按钮(重试/返回)

| 错误类型 | 标题 | 描述 |
|----------|------|------|
| API超时 | 请求超时 | 分析耗时较长，请稍后重试或减少样本数量 |
| 网络错误 | 网络连接异常 | 请检查网络后重试 |
| AI服务不可用 | AI服务暂不可用 | 当前无法生成回答，可稍后再试 |

## 4. 成功反馈设计

- 对比完成：顶部轻提示 + 绿色完成状态点
- 导出成功：message.success + 按钮旁2秒"已导出"
- 会话创建：新会话项高亮2秒 + 滚动定位

### 间距
- 4px/8px/12px/16px/24px/32px 递进

- 高度32px（普通）/ 36px（CTA）
