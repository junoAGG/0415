# 投研助手 - 智能分析模块测试计划

> 任务编号：Task #7  
> 版本：v1.0  
> 编写日期：2026-04-16  
> 适用范围：智能分析模块优化后的功能验证

---

## 一、测试目标

### 1.1 质量目标
- 确保对比分析、AI问答、数据导出、会话管理等核心功能稳定可用
- 验证错误处理机制（超时、中断、网络异常）正确生效
- 确认前端组件拆分后的交互体验保持一致
- 保证原有研报上传/解析/列表功能未受影响（回归测试）

### 1.2 测试范围

**在测功能：**
- 对比分析功能（多维度对比、场景切换）
- 流式问答功能（SSE流式响应、中断/重试、会话切换）
- 数据导出功能（Markdown/JSON/CSV格式、会话导出）
- 会话管理功能（搜索、标签CRUD、分页、分组显示）
- 错误处理与容错（API超时、网络中断、非法输入）
- 组件交互（空状态、骨架屏、折叠/展开、KPI卡片）

**不在测试范围：**
- 研报上传/解析的具体算法准确性（已在其他任务覆盖）
- AI生成内容的质量评估（由业务方人工审核）
- 性能压力测试（并发用户数 > 50 的场景）

---

## 二、测试策略

### 2.1 测试层级

| 层级 | 覆盖范围 | 测试方法 |
|------|----------|----------|
| 单元测试 | 工具函数、类型定义、状态转换 | 白盒测试 |
| 集成测试 | API接口、前后端数据流、组件交互 | 灰盒测试 |
| 端到端测试 | 完整用户流程、跨功能场景 | 黑盒测试 |
| 回归测试 | 原有研报功能、数据一致性 | 黑盒测试 |

### 2.2 测试方法

- **功能测试**：基于用例的正面/负面场景验证
- **边界测试**：空数据、超长文本、极限参数
- **异常测试**：网络中断、服务不可用、超时场景
- **兼容性测试**：主流浏览器（Chrome/Firefox/Safari/Edge）

## 五、自动化测试方案

### 5.1 框架选型

| 层级 | 推荐框架 | 理由 |
|------|----------|------|
| 前端单元测试 | Vitest + React Testing Library | 与Vite项目集成好，React组件测试友好 |
| 前端E2E测试 | Playwright | 支持多浏览器，SSE测试能力强 |
| 后端API测试 | pytest + requests | Python生态，与后端技术栈一致 |
| 性能测试 | k6 / Locust | 支持SSE和WebSocket测试 |

### 5.2 自动化范围

**适合自动化：**
- API接口测试（对比、问答、导出、会话CRUD）
- 组件渲染测试（EmptyState、Skeleton、KPI卡片）
- 错误处理流程测试
- 回归测试用例

**不适合自动化：**
- AI生成内容质量验证（需人工判断）
- UI视觉效果测试（需视觉回归测试工具）
- 复杂交互流程（维护成本高）

### 5.3 实施计划

| 阶段 | 目标 | 时间线 |
|------|------|--------|
| 第一阶段 | API自动化测试覆盖（pytest） | 1周 |
| 第二阶段 | 核心组件单元测试（Vitest） | 1周 |
| 第三阶段 | 关键流程E2E测试（Playwright） | 1周 |
| 第四阶段 | CI/CD集成 | 3天 |
