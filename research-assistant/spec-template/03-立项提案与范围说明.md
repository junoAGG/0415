## 1. 背景与问题

### 痛点1：信息分散
- 研报、新闻、公告、股票数据分布在多个系统（Wind / 同花顺 / 内部系统）
- 无统一入口，需频繁切换系统
- 来源：用户访谈（S-02、S-03）

### 痛点2：研报阅读效率低
- 每天需阅读10+份研报，每份30–50页
- 平均耗时2–3小时，关键信息提取依赖人工
- 来源：用户访谈（S-02、S-03）

### 痛点3：关键信息提取不完整
- 评级、目标价、核心观点分散在不同段落/表格
- 易遗漏，且难结构化沉淀
- 来源：会议纪要（S-01）

### 痛点4：多研报对比困难
- 不同券商格式差异大
- 需人工复制到Excel整理，效率低
- 来源：用户访谈（S-02、S-03）

### 痛点5：股票数据查询割裂
- 阅读研报时无法直接查看对应股票数据
- 需切换系统查询行情与指标
- 来源：会议纪要（S-01）

---

## 2. 提案概要

在投研助手应用中，提供**"研报自动解析 + 智能问答 + 多研报对比 + 数据联动"**能力，帮助分析师快速获取关键信息。

- 系统支持PDF/HTML研报解析，自动抽取评级、目标价、核心观点，并结构化展示
- 通过基于RAG的问答能力，实现对研报内容的自然语言查询，并支持引用原文定位
- 同时提供多研报自动对比功能，自动生成对比表，减少人工整理成本
- 股票数据支持独立查询和研报联动两种模式，实现数据无缝衔接
- 本期不支持本地部署模型，而是支持使用外部模型的方式进行处理。

## 3. 范围边界

### 3.1 做（范围内）

| 序号 | 功能 | 说明 |
|:---:|------|------|
| 1 | 研报解析 | 支持PDF/HTML上传，自动提取结构化信息（评级、目标价、核心观点） |
| 2 | 研报问答 | 基于单篇/多篇研报的自然语言问答，支持引用原文 |
| 3 | 多研报对比 | 同一公司多研报自动对齐并生成对比表 |
| 4 | 股票数据汇总查询 | 在研报阅读界面展示基础行情（价格、PE等），同时支持单独搜索查看股票信息 |
| 5 | 基础用户功能 | 研报列表、阅读、收藏、标注 |

### 3.2 不做（范围外）

| 序号 | 功能 | 原因 |
|:---:|------|------|
| 1 | 多租户/跨机构数据权限体系 | 复杂权限控制，本期聚焦核心功能 |
| 2 | 高级量化分析功能 | 如因子回测、策略构建，超出本期范围 |
| 3 | 实时交易或交易系统集成 | 合规风险高，仅做数据展示 |
| 4 | 全量舆情分析平台 | 仅做基础聚合，不做复杂情绪建模 |

## 4. 成功标准（可度量）

### 标准1：主路径可走通
- 用户在3分钟内完成"上传研报 → 自动解析 → 提问 → 查看答案（带引用）"全链路

### 标准2：阅读效率提升
- 用户获取"评级 + 目标价 + 核心观点"耗时 ≤ 1分钟
- 对比传统阅读（2–3小时）效率提升 ≥ 70%

### 标准3：信息提取准确率
- 评级/目标价提取准确率 ≥ 90%（抽样验证）

### 标准4：多研报对比效率
- 用户在1分钟内生成3份研报对比表
- 对比效率提升 ≥ 80%（相对人工Excel）

---

## 5. 风险（简表）

### 风险1：外部模型能力不稳定
- **影响**：问答质量下降或不可用
- **应对**：
  - 接入多模型供应商（OpenAI / 智谱 / 文心）
  - 降级策略：RAG → 关键词检索 → 原文定位

### 风险2：PDF解析效果不稳定（格式复杂）
- **影响**：关键信息提取不准确
- **应对**：
  - 多解析方案（PDF解析 + OCR + 表格识别）
  - 关键字段规则兜底（正则+模板）

### 风险3：数据源接口限制（行情/财务数据）
- **影响**：股票数据联动能力受限
- **应对**：
  - 接入多数据源（Tushare / 内部API）
  - 本地缓存常用数据
